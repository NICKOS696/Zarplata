# 🚀 Быстрый старт

## Шаг 1: Установка зависимостей

### Backend (Python)
```powershell
cd backend
pip install -r requirements.txt
```

### Frontend (Node.js)
```powershell
cd frontend
npm install
```

## Шаг 2: Настройка окружения

Создайте файл `backend/.env` на основе `.env.example`:

```env
DATABASE_URL=sqlite:///./sales.db
TELEGRAM_BOT_TOKEN=your_bot_token_here
SECRET_KEY=your_secret_key_here
DEBUG=True
CORS_ORIGINS=http://localhost:5173
```

## Шаг 3: Инициализация базы данных

```powershell
cd backend
python init_db.py
```

Это создаст базу данных и добавит тестовые данные:
- 1 супервайзер
- 4 менеджера
- 4 бренда
- 3 типа KPI
- Планы и факты продаж
- Табель посещаемости
- Правила расчета зарплаты

## Шаг 4: Запуск приложения

### Запустите Backend (в одном терминале):
```powershell
cd backend
uvicorn main:app --reload --port 8000
```

Backend будет доступен на: http://localhost:8000
API документация: http://localhost:8000/docs

### Запустите Frontend (в другом терминале):
```powershell
cd frontend
npm run dev
```

Frontend будет доступен на: http://localhost:5173

## Шаг 5: Начало работы

1. Откройте браузер: http://localhost:5173
2. Вы увидите дашборд с тестовыми данными
3. Начните работу:
   - **Сотрудники** - управление командой
   - **Планы продаж** - установка планов
   - **Факты продаж** - ввод фактических данных
   - **Табель** - учет посещаемости
   - **Зарплата** - расчет зарплаты
   - **Импорт** - загрузка данных из 1С
   - **Настройки** - бренды, KPI, правила

## 📊 Основные функции

### 1. Управление сотрудниками
- Добавление менеджеров и супервайзеров
- Установка фиксированной зарплаты
- Привязка к Telegram (для будущих уведомлений)

### 2. Планирование продаж
- Установка планов по сотрудникам
- Разбивка по брендам и KPI
- Гибкие периоды планирования

### 3. Учет продаж
- Ручной ввод продаж
- Импорт из HTML файлов 1С
- Фильтрация по сотрудникам, брендам, датам

### 4. Табель посещаемости
- Учет рабочих дней
- Отметка присутствия/отсутствия
- Учет часов работы
- Импорт из 1С

### 5. Расчет зарплаты
- Автоматический расчет по правилам
- Фиксированная часть (с учетом посещаемости)
- Мотивационная часть (% от продаж)
- Бонусы за перевыполнение
- Штрафы за прогулы

### 6. Гибкая настройка
- Добавление/удаление брендов
- Настройка типов KPI
- Правила расчета зарплаты:
  - Процент от продаж
  - Ступенчатая система
  - Фиксированные бонусы

## 🔧 Настройка правил зарплаты

### Пример 1: Процент от продаж
```json
{
  "base_percent": 3,      // Базовый процент
  "bonus_percent": 2,     // Дополнительный процент при перевыполнении
  "threshold": 100        // Порог перевыполнения (%)
}
```

### Пример 2: Ступенчатая система
```json
{
  "tiers": [
    {"from": 0, "to": 80, "percent": 1},
    {"from": 80, "to": 100, "percent": 3},
    {"from": 100, "to": 120, "percent": 5},
    {"from": 120, "to": 999, "percent": 7}
  ]
}
```

### Пример 3: Фиксированный бонус
```json
{
  "threshold": 100,       // Порог выполнения (%)
  "amount": 10000         // Сумма бонуса
}
```

## 🐛 Решение проблем

### Backend не запускается
- Проверьте, что установлены все зависимости: `pip install -r requirements.txt`
- Проверьте файл `.env`
- Убедитесь, что порт 8000 свободен

### Frontend не запускается
- Проверьте, что установлены зависимости: `npm install`
- Убедитесь, что порт 5173 свободен
- Проверьте, что backend запущен

### Ошибка импорта из 1С
- Проверьте формат HTML файла
- Убедитесь, что сотрудники добавлены в систему
- Имена сотрудников в файле должны совпадать с именами в системе

### Неправильный расчет зарплаты
- Проверьте правила расчета в настройках
- Убедитесь, что есть планы продаж
- Проверьте факты продаж за период
- Проверьте табель посещаемости
